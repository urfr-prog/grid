2.1 Dé-culbutage des faces paires FAIT & VALIDÉ

Diagnostic device (mesuré, pas déduit). Le carrousel tourne sur X : chaque face a mount.rotation.x = -ang avec ang = f·(2π/NBOARDS). Les faces où |cos(ang)|≈1 (faces paires : brd 0 et brd 2, = « plateaux 1 et 3 » dans le langage de Gregory) sont montées tête-bêche → r0/r4 inversés à l'écran vs faces impaires. Sondes device décisives : brd0 cos=+1 → r4 EN HAUT ; brd1 cos=0 → r4 en bas ; brd2 cos=−1 → r4 EN HAUT ; brd3 cos=0 → r4 en bas.

Solution = mapping coordonnées (approche B), pas rustine geste. On inverse la rangée uniquement sur les faces culbutées, dans le mapping logique→écran. Le vertical seul est miroité ; x / colonnes / icônes / tapis horizontal intacts. Le geste suit alors automatiquement.

function faceFlipped(f){ const ang=f*(2*Math.PI/NBOARDS); return Math.abs(Math.cos(ang))>0.5; }
function cellToLocal(col,row,face){
  if(face!=null && faceFlipped(face)) row=(ROWS-1)-row;   // dé-culbute : rangée miroir
  return { x:(col-(COLS-1)/2)*CELL_W, y:((ROWS-1)/2-row)*CELL_W };
}

Le 3ᵉ paramètre face est optionnel (rétro-compatible). Il a été branché à TOUS les sites qui posent/lisent des coordonnées :

  • Placement : placePiece(p) → p.face (donc gravité / fusion / swap en héritent, ils passent tous par placePiece).
  • Portes : syncPortes (boucle f), commutePorte (boucle f), proxy porte (B.currentBoard).
  • Picking : cellAt applique la même inversion (if(faceFlipped(B.currentBoard)) row=(ROWS-1)-row;) — symétrique de cellToLocal.
  • Slide : les 4 cellToLocal de applySlide reçoivent B.currentBoard.

2.2 Tapis-roulant vertical réaccordé sur faces culbutées FAIT & VALIDÉ

Le piège qui a coûté plusieurs allers-retours. Trois repères coexistent et devaient TOUS être cohérents sur une face culbutée :
  1. Position de départ (base.y) : dé-culbutée via cellToLocal(...,face). ✔
  2. Incrément du geste (off) : doit être miroité, sinon le doigt et la pièce divergent. → applySlide(faceFlipped(B.currentBoard) ? -pxToWorldV(dy) : pxToWorldV(dy)).
  3. Bornes de wrap (maxY/minY) : cellToLocal échange les y de row 0 et ROWS−1 sur face culbutée → prendre Math.max/Math.min EXPLICITES, sinon la pièce saute à la mauvaise case.
  4. Commit logique (commitSlide) : dragOff est en repère ÉCRAN miroité → ré-inverser steps sur face culbutée, sinon le visuel glisse bien mais la grille se réécrit à l'envers (« désorganisation »). → steps = faceFlipped(B.currentBoard) ? -rawSteps : rawSteps; (axe vertical uniquement).
Règle à retenir : cellToLocal traduit logique→écran pour les positions ; il faut le pendant pour le mouvement (off) ET pour le commit (steps). Les trois, sinon divergence.

3. ★ FUSION INTER-BANC — livrée cette session P5-1 / IB VALIDÉ

3.1 Règle de game design (verrouillée avec Gregory)

  • Magnet, pas contact de bord : deux dominos qui se font face à row égal, même non collés au bord, s'attirent à travers le gap (couloir vide requis dans chaque grille ; le gap est vide par nature).
  • Percuteur = le domino qui a bougé EN DERNIER (_settle le plus récent). Il TRAVERSE le gap.
  • Percuté = l'immobile. Il reste ; le résultat fusionné atterrit sur SON banc.
  • Déclenchement AUTO : en fin de checkFusions, après les fusions/aimants mono-banc, quand rien d'autre n'est possible.

3.2 Implémentation

Détection — interBankPairs() (réparée P5-0b) : faces COURANTES de chaque banc (banks[b].currentBoard), logique magnet (pièce la + à droite de g0 ↔ la + à gauche de g1, couloir vide dans chaque grille), match via magnetMatch. Renvoie {row, i0, i1, icon}.

Action — tryInterBankFusion() (juste après interBankPairs), branchée dans checkFusions par if(tryInterBankFusion()) return; après tryMagnet :

1. pairs = interBankPairs(); prendre pairs[0].
2. percuteur = _settle le + récent (p0 vs p1) ; ex-aequo → p0 percute (défaut).
3. dstBank = banc du PERCUTÉ ; dstIdx = case adjacente à la victime, côté gap
   (p0 percute → i1−1 ; p1 percute → i0+1). Garde-fous : dstIdx in-range, même row, case libre.
4. retirer percuteur de son board source ; l'écrire dans dstGrid[dstIdx] ;
   hitter.key/face/layer ← banc du percuté.
5. faceGroups[dstKey].attach(hitter.mesh)  // préserve la position MONDE → départ visuel correct
6. setActiveBank(dstBank) ; animer le glissement (swapAnim from→to via placePiece) ;
   after: performFusion(iLeft,iRight,'h')   // réutilise toute la séquence move/fade/appear + montée d'état
Verrou réentrance : _ibBusy + banks[0].busy = banks[1].busy = true pendant l'opération.
Pourquoi ça réutilise tout l'existant. Une fois le percuteur co-localisé dans la grille du percuté, les deux pièces sont adjacentes dans un seul board → performFusion(...,'h') standard s'applique sans modification (faces résultantes via getRemainingFace, état +1 cran, anim, propagation portes). La seule nouveauté structurelle est la TRAVERSÉE (réattachement de mesh + écriture cross-banc).
